The Perfect Shell

Why cmd.exe, sh and bash are not perfect shells:
  • Limited and syntactically awkward programming (scripting) possibilities 
Why perl, python, ... are not perfect shells:
  • Too verbose to run external programs
  • Too verbose to setup pipes
  • Too verbose to switch directories
What would the perfect shell look like for me:
  • Full programming language syntax (preferably Python or Groovy or Beanshell)
  • Very succinct syntax for program invocation and pipelining
  • Very succinct syntax for changing the current working directory
  • Built-in and succinctly invocable functionality for many commonly used Unix utilities: ls, find, grep, sed, date, dirname, basename, top, ps, pwd, df, du
  • Built-in functionality for switching directories with full directory history
  • Built-in functionality for selecting files based on re's or glob's
  • Built-in and context-sensitive tab-completion for: file-names, program-names, function-arguments and of course: command-history
  • Built-in functionality for starting tasks in the background
What could also be cool:
  • Syntax-coloring
  • Web-based shell
  • Collapsable output and error output from programs
  • Input forms for gathering input
  • Good support for cut, copy & paste (also for column mode)

Possible Syntax


Command history: cmd<cmd-nr>
Output history: out<cmd-nr>
File selection: based on globs: f<*.txt>  based on regexes: fr<^f.*\.txt>
Command invocation: 
  Simple commands: <date +%Y-%m-%d>
  Piping (| pipes stdout, || pipes stderr, ||| pipes stdout and stderr): <ls -lh> | <sed -e "s/my/you/g"> ||| <uniq -c> 
  Redirection (< input from right, > output to right): <cat f<file-1.txt>> | <sed -e "s/my/yu/g"> > f<file-2.txt>
